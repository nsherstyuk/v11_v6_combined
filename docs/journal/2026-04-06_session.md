## Decision Made Late Session: Hybrid Backtest Approach

**User confirmed:** Two-stage hybrid approach to backtesting.

### Stage 1 — Parameter Grid Search (No LLM)

- Run DarvasDetector + ImbalanceClassifier over historical bars, many parameter combos
- No LLM calls — fast, deterministic, repeatable
- Simulate trades: entry at breakout_price, SL at box boundary, time stop
- Metrics: signal count, win rate, avg R:R, max drawdown, Sharpe
- Output: ranked parameter sets per instrument

### Stage 2 — LLM Filter Validation (Best Params + Grok)

- Take top 1–3 param sets from Stage 1
- Re-run, calling Grok for every signal with full `SignalContext` (including historical timestamp)
- Grok can reason about calendar from timestamps: "first Friday = NFP", "mid-Dec = thin liquidity"
- Grok has same limitations in backtest as live: no hindsight, no exact schedules beyond training cutoff, no real-time data
- Compare filtered vs unfiltered results → answers "does Grok actually help?"

### Why Hybrid?

- Grid search produces thousands of signals → calling LLM for each is slow, expensive, non-deterministic
- Stage 1 finds good Darvas params, Stage 2 validates the LLM value-add
- Grok's calendar reasoning works on historical timestamps the same way it works live

### Backtest Files to Build

```
v11/backtest/
├── data_loader.py        # Load historical bars from CSV or IBKR
├── grid_search.py        # Stage 1: Darvas param optimization (no LLM)
├── llm_validation.py     # Stage 2: Best params + Grok filter
├── simulator.py          # Trade simulation (entry, SL, time stop, PnL)
└── metrics.py            # Win rate, R:R, drawdown, Sharpe, Calmar
```

## External Review Findings (Late Session)

An external LLM reviewed all three design documents. Three critiques raised:

### Critique #1: Latency Gap (🔴 FIXED)

Price can move $2–$4 on XAUUSD during Grok's 2–3 sec response time.

**Action taken:** Implemented **slippage ceiling** in `live_engine.py _handle_signal()`.
- New param: `max_entry_drift_atr = 0.5` in LiveConfig
- After LLM approves, checks `abs(current_price - breakout_price)` vs `0.5 × ATR`
- If exceeded → trade aborted, logged as `ENTRY DRIFT ABORT`
- Also added `_last_price` tracking in InstrumentEngine so current price is always known

**Files changed:** `v11/config/live_config.py` (added param), `v11/live/live_engine.py` (added check + price tracking)

### Critique #2: Calendar Hallucination (🟡 Deferred)

Grok knows general FOMC/NFP/CPI schedule but not moved meetings or surprise announcements.

**Action:** Documented. Already scoped as Decision #4. Must wire up external calendar API (ForexFactory, TradingEconomics, FCS) before going live with real money. Fine for paper trading + backtesting.

### Critique #3: Zoomed-In Bias (🟡 Deferred)

200 × 1-min bars = ~3.3 hours. No daily/4H context for Grok to see higher-timeframe resistance.

**Action:** Documented. Schema already has `daily_bars: Optional[List[BarData]]`. Need to wire up IBKR daily bar fetch or external data source. Must do before live trading.

### Critique on Watchlist Strategy (Out of Scope)

Applies to Swing Agent (Project 1), not V11. Noted, no action.
