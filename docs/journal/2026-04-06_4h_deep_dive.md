## Recommended Strategy Architecture

Based on ALL research across 5 sessions, the system should be a **dual-signal portfolio**:

### Signal 1: Darvas Box Breakout (High Quality, Low Frequency)

```
1-min bars -> DarvasDetector (tc=20, bc=12, mxW=3.0, brk=3)
    -> CONFIRMING volume check
    -> 60-min SMA(50) direction alignment
    -> Entry at breakout price
    -> SL at box boundary
    -> TP at entry + risk * 2.0
    -> Trail10@60 after 1 hour
    -> ~15 trades/year, +0.175 AvgR
```

### Signal 2: 4H Level Retest (Moderate Quality, Higher Frequency)

```
4H bars -> Swing high/low detection (lb=10, rb=10)
    -> Levels persist up to 72 hours
    -> 1-min bar breaks level (initial break, no entry)
    -> Wait 10-30 bars for pullback to level
    -> 1-min bar rebreaks level
    -> CONFIRMING volume check
    -> 60-min SMA(50) direction alignment
    -> Entry at rebreak price
    -> SL at level +/- 0.3 ATR
    -> TP at entry + risk * 2.0
    -> ~22 trades/year, +0.135 AvgR
```

### Shared Components

- **60-min SMA(50)** — direction filter for both signals
- **ImbalanceClassifier** — volume confirmation for both signals
- **BarAggregator** — tick -> 1-min bar conversion
- **TradeManager** — bracket orders, fill tracking, position reconciliation
- **IBKRConnection** — market data and execution
- **Risk limits** — daily loss cap, position sizing

### Do We Need the LLM?

**Short answer: No, for the core edge. Possibly yes, for refinement.**

The mechanical system produces ~5.6R/year without any LLM involvement. The alpha comes from:
1. 60-min SMA(50) direction alignment (the primary edge)
2. Structural signals (Darvas consolidation breakout, 4H level retest)
3. Volume confirmation (CONFIRMING filter)

The LLM (Grok) could add value as an **optional quality filter**:
- Reject signals near scheduled news (FOMC, NFP) — avoid volatility spikes
- Assess breakout quality ("is this a clean break or choppy?")
- Higher-timeframe context beyond SMA (daily trend structure, weekly levels)
- Pattern quality scoring (clean consolidation vs messy range)

But the risk of adding the LLM is:
- Latency (2-3 sec) could cause missed entries or worse fills
- Non-deterministic — harder to backtest and validate
- Additional failure mode (API down, bad JSON, hallucination)
- The edge is already thin (+0.135 to +0.175 AvgR) — LLM rejection of good trades could hurt more than acceptance of bad trades helps

**Recommendation:** Build the mechanical system first. Paper trade it. Add Grok as an optional layer later, comparing filtered vs unfiltered performance in live conditions. This is exactly what the two-stage hybrid approach in V11_DESIGN.md prescribed.
